# crtfyStudent Pricing Model Assumptions

## Pricing Dimensions

Institution size:
- Small: fewer than 2,500 annual applicants
- Mid-market: 2,500 to 15,000 annual applicants
- Enterprise: more than 15,000 annual applicants

Applicant volume:
- Annual applicant count
- Transfer applicant count
- Transcript/document volume
- Peak-season concurrency

Module scope:
- Operations
- Decision Intelligence
- Yield And Handoff
- Platform

Integration complexity:
- No live integration, import/export only
- One CRM or SIS connector
- Multiple connectors
- Bidirectional sync
- Custom object and field mapping

Implementation complexity:
- Standard templates and roles
- Program-specific checklist/routing rules
- Multiple campuses or departments
- Graduate or faculty review workflows
- Custom reporting dimensions

## Packaging Assumptions

Package 1: Operations
- Base subscription priced by applicant volume.
- Includes core admin, Student 360, checklist, document queue, today's work, and operational reporting.
- Implementation assumes standard admissions roles and one checklist template family.

Package 2: Decision Intelligence
- Add-on priced by decision packet volume and transfer/transcript complexity.
- Includes trust, duplicate, transfer evidence, decision studio, recommendation workflow, and release controls.
- Implementation assumes decision roles, release gates, and review workflow tuning.

Package 3: Yield And Handoff
- Add-on priced by admitted-student volume.
- Includes yield queue, deposit/melt, milestones, handoff, and sync error workflow.
- Implementation assumes milestone templates and office ownership setup.

Package 4: Platform
- Platform tier priced by connector count, environments, and implementation complexity.
- Includes connector status, mappings, sync observability, implementation readiness, and benchmark reporting.
- Implementation assumes IT participation and integration validation.

## Implementation Fee Assumptions

Low complexity:
- One package
- One tenant
- Standard roles
- No live integration
- 30-day implementation path

Medium complexity:
- Two or three packages
- One or two live integrations
- Program or population-specific checklist logic
- 60-day implementation path

High complexity:
- All packages
- Multiple integrations
- Multi-campus or graduate review
- Custom reporting dimensions
- 90-day implementation path

## Value Metrics To Track

- Reduction in incomplete application aging
- Faster document indexing and exception resolution
- Faster complete-to-decision cycle time
- Higher admitted-to-deposit conversion
- Lower melt risk for admitted students
- Reduced manual CRM/SIS lookup time
- Improved auditability and governance
